# team-roles — 会社型の役割分担フロー

Pachi Tracker の開発を「会社の組織」に見立て、役割を分けて安全に進めるためのスキル。
1人のモデルが全部やるのではなく、**メインエージェントが分析・指揮し、サブエージェント（現場）が実装し、メインエージェントが検収・評価する**。

## 組織図（誰が何をするか）

### メインエージェント（司令塔）
- **このセッションを操作しているモデル自身が務める。**
- 役割:
  1. タスクの分析・要件定義・根本原因の特定
  2. 設計判断（state構造・データフロー・既存設計との整合）を**先に確定**する
  3. 実行モデル（Sonnet / Opus）の選定（下の選定基準に従う）
  4. サブエージェントへ**明確な仕様書**を渡して委譲する
  5. 成果物の検収（lint / build / test の確認、差分レビュー）
  6. 評価と改善点の指摘。必要なら差し戻し、または自身で最終修正
  7. コミット・push・HANDOVER.md / Notion への記録
- メインエージェントはロジックの心臓部（`src/logic.js`・`src/evDecision.js`）を守る最後の砦。

### サブエージェント（実行担当）
- **Sonnet または Opus**（メインエージェントがタスクごとに選ぶ）。`Agent` ツールで起動する。
- 役割:
  1. メインエージェントの仕様書どおりに実装する
  2. `npm run lint` / `npm run build` /（あれば）テストを**自分で通す**
  3. **コミット・push はしない。** 差分を作業ツリーに残してメインエージェントへ報告する

## 実行モデルの選定基準（メインエージェントが判断）

| 判断軸 | Sonnet 向き | Opus 向き |
|---|---|---|
| 仕様の明確さ | 仕様が確定・範囲が限定的 | 設計判断や不確実性が残る |
| 性質 | UI実装・既存パターン流用・定型作業 | アルゴリズム/計算ロジック近傍・移植・統合 |
| 影響範囲 | 1〜数ファイルで完結 | 複数ファイル横断・state整合が複雑 |
| リスク | 金銭計算に触れない | logic.js 近傍・境界値検証が必要 |

迷ったらメインエージェントが分析で不確実性を潰してからSonnetに渡すのが基本。設計判断が残るならOpus。

## 進め方（手順）

1. **メインエージェントが分析**: 関連コードを読み、根本原因と設計方針を確定する。
   ユーザーへ診断結果と設計判断を一度提示する。
2. **モデル選定**: 上の基準でSonnet/Opusを選び、理由を述べる。
3. **委譲**: `Agent` ツールでサブエージェント（`model: sonnet` または `opus`）を起動。
   仕様書には必ず以下を含める:
   - 作業ディレクトリと対象ブランチ
   - CLAUDE.md の厳守事項（logic.js/evDecision.js 変更禁止、日本語UI、44px、ついで改修禁止 等）
   - 解決すべき問題と**確定済みの設計**
   - 参考にすべき既存コード箇所
   - 完了前に lint/build/test を通すこと
   - **コミット・pushはしない**指示
   - 報告フォーマット（CLAUDE.md「変更の報告」準拠）
4. **報告**: サブエージェントがメインエージェントへ成果物を報告する。
5. **検収・評価**: メインエージェントが差分をレビューし、lint/build/test を自分でも確認。
   評価（良かった点・改善点）を述べ、必要なら修正する。
6. **記録と確定**: メインエージェントがコミット・push し、HANDOVER.md に
   「実装: <サブエージェントのモデル> / 検収・修正: メインエージェント」の形で記録する。

## 守るべき前提（このリポジトリ固有）

- セッション開始時の必須手順（`docs/HANDOVER.md`・`CLAUDE.md` 確認）はメインエージェントが先に済ませる。
- `src/logic.js`・`src/evDecision.js`・保存データ構造・計算式は変更禁止。
- UIラベルは日本語のみ。タップ領域44px以上。横スクロール前提UI禁止。
- 「ついでの別機能改修」は禁止。メインエージェントが承認したスコープのみ。
- コミットメッセージは日本語。HANDOVER.md は最新を上に追記する。

## 報告テンプレート（サブエージェント → メインエージェント）

1. 変更したファイル
2. 変更内容（何を・どこに）
3. 操作ステップ数への影響
4. `src/logic.js`・`src/evDecision.js` を変更していないことの明言
5. 保存データ構造・計算式を変更していないことの明言
6. `npm run lint` の結果
7. `npm run build` の結果
8. テストの結果
9. 残課題・懸念
